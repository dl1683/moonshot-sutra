The ideas with the best any-scale grounding are the ones that come with real invariants: contraction and norm control, multiresolution structure, and sparse local message passing with spatial coupling. Those are the three proposals I would actually trust across scale.

1. **Contractive Hyperspherical Core**
Use normalized state geometry as the backbone: keep every token state on a sphere, use cosine-based interactions, make recurrent updates tangent to the sphere, and force every write/update to be a bounded convex move. Concretely: orthogonal or unitary transport for state propagation, bounded residual step sizes, and no raw-magnitude-dependent routing. This is scale-invariant because the governing quantities are angles, spectral norms, and contraction constants, not hidden width. `d=128` and `d=768` are the same dynamical system class. This is the best immediate replacement for fragile warm-start logic, because any new module can be added through an exact identity homotopy.

2. **RG/MERA Stage Pyramid**
Replace the flat recurrent workspace with an explicit multiscale graph: token nodes, phrase nodes, segment nodes, document nodes. Use wavelet or lifting-style up/down passes so every recurrent cycle does coarse-grain, communicate, then refine. Share the same local update rule across scales. This is scale-invariant by construction because the architecture grows by adding levels, not by changing the primitive. Width only changes channel count; sequence scale is handled by topology. The theoretical support here is unusually strong in the synthesis: wavelets/MRA, renormalization group, MERA/holography, and coarse-to-fine self-organization all point to the same answer.

3. **Spatially Coupled Predictive-Coding Graph**
Stop broadcasting dense hidden state. Make communication channels carry only sparse residuals or prediction errors over a fixed sparse graph with local edges plus expander-style long-range edges. Think LDPC belief propagation, not attention imitation: iterative local updates, boundary seeding, spatial coupling, fixed packet budget per node. This is scale-invariant because global performance depends on graph expansion and residual sparsity, not ambient dimension. Hidden size can change, but the communication law stays the same. It also matches the strongest cross-domain convergence in the synthesis: predictive coding, BP on sparse codes, spatial coupling, expander graphs, and stigmergic shared-state systems.

If I had to rank them: the safest foundation is **Contractive Hyperspherical Core**, the strongest long-context architecture is **RG/MERA Stage Pyramid**, and the best replacement for brittle scratchpad/router heuristics is **Spatially Coupled Predictive-Coding Graph**.

If you want the highest-confidence direction, combine `1 + 3`: geometric contraction for stability, sparse error-passing for communication. That is the cleanest path to something that does not care whether the internal width is `128` or `768`.
