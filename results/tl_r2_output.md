- Norm everywhere is `SSRMS(x) = g_l * x / sqrt(mean(x^2)+1e-6)`, one scalar `g_l` per block, not per channel. DyT is off the mainline.
- The `Δ` block is `u_t = SSRMS(h_t)`, `q,k,v,α,β = W_* u_t`, `S_t = α_t ⊙ S_{t-1} + β_t (k_t ⊗ v_t)`, `m_t = W_o(S_t q_t)`, `r_t = h_t + m_t / sqrt(12)`, `h_{t+1} = r_t + FFN(SSRMS(r_t)) / sqrt(12)`. The explicit `1/sqrt(12)` residual scale is the Round 2 fix for residual-stream blowup.
- The `A` block is unchanged GQA+RoPE: `r_t = h_t + Attn(SSRMS(h_t)) / sqrt(12)`, then the same scaled SwiGLU FFN. Attention stays sparse because its job is exact retrieval and synchronization, not bulk compute.
- Knowledge absorption is architectural, not bolted on. Pooled state feeds ports for `decoder=Qwen3-0.6B`, `ssm=Mamba-790M`, `hybrid=Granite-4.0-Micro`, `semantic=EmbeddingGemma-308M`, `code=CodeBERT`. One structural family is active per generic batch; semantic runs every 4th generic batch; code teacher is always on for code batches. Hard-doc top-32 logits are cached only for the hardest `2%` of windows.
- Memory stays additive and delayed: bigram/trigram hash tables on CPU RAM, zero-init projection, gate bias `-2.0`, injected only after block `8`, and only after the revised O4 stack proves positive.
- Optimizer stack is exact: Muon on all dense matrix weights with `lr=2e-2` scout / `1.5e-2` main, `momentum=0.95`, `weight_decay=0.1`, `ns_steps=5`; AdamW on biases, scalar gates, embeddings, and norm scalars with `lr=3e-4`, `betas=(0.9,0.95)`, `wd=0`; SparseAdam on memory tables with `lr=6e-4`.
