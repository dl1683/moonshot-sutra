# Sutra (सूत्र)

**Deriving intelligence from first principles — proving that better mathematics beats more parameters.**

> *"You don't need a data center to be intelligent. You just need better geometry."*

Named after the Sanskrit tradition of compressing vast knowledge into minimal, precise threads. Panini encoded all of Sanskrit grammar in ~4,000 sutras. We're encoding intelligence the same way.

---

## The Axiom We Challenge

**"Frontier AI requires frontier scale."** Every competitive model today is built by scaling existing architectures with more parameters, more data, more compute. We believe a system designed from first principles — derived from mathematics, not copied from existing architectures — can achieve frontier-level text intelligence at a fraction of the parameter count.

## The Goal

Build a sub-4B parameter intelligent text system from scratch that:

- **Derives from mathematics**, not from copying existing architectures
- **Designs its own representations** — tokenization, embeddings, data encoding all purpose-built
- **Trains from scratch** — no fine-tuning someone else's foundation model
- **Runs on edge hardware** — phones, laptops, embedded devices
- **Outperforms models of equal or larger size** on reasoning, strategy, synthesis, and critical analysis
- **Proves Intelligence = Geometry** — that mathematical structure beats brute-force scale

The answer might not be a neural network. The search space includes all of ML, biology, mathematics, physics, and any system that processes information efficiently.

## The Approach

Five irreducible operations of intelligence, derived from first principles:
1. **Compress** — prediction IS compression (Shannon 1948)
2. **Compose** — combine known concepts into novel structures (algebraic)
3. **Abstract** — collapse equivalent forms into shared representations (quotient space)
4. **Reason** — chain inference steps of variable depth (iterated function composition)
5. **Select** — choose from exponential continuation space (search/optimization)

Everything else is implementation detail. We derive the minimal system that supports all five.
